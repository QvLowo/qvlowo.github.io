Hexo Blog教學(三)：將你的網站手動部署到 GitHub Pages

發表於2025-01-23|更新於2025-09-17|learning-guides
|總字數:968|閱讀時間:3分鐘|瀏覽量:

上一篇：建立你的第一個blog與更換漂亮的主題

前言

上一篇，我們提到了如何使用hexo架設個人blog，但檔案都還是在本地，這代表只有你的電腦才看得到blog網頁，
那要怎麼讓別人也看得到你的blog呢？
這時候就要來聊聊部署了！
簡單來說，部署就是將你本地的網站檔案，發布到一個公開的網路伺服器上。
本篇教學會帶你使用完全免費的GitHub Pages服務，完成你的第一次網站部署！

第一步：建立 GitHub Repository

首先，我們需要到GitHub上建立一個專門存放網站檔案的Ropository。
這個Repository的名稱有固定格式，一定要按照需求，不要自定義。

Repository名稱：<你的帳號>.github.io

請先登入你的GitHub帳號，並建立一個新的Repository。

建立時有 2 個重點要注意：

  1. 帳號大小寫：不論你的 GitHub 帳號原始大小寫為何，Repository的名稱務必全部使用小寫。例如，帳號是 MyName，那名稱就要設為 myname.github.io。
  2. 公開：Repository必須設定為Public，這樣GitHub Pages的服務才能讀取你的檔案，並將網站公開給所有人瀏覽。

如果已經建立過同名的Repository，系統會出現紅色警告是正常的(像圖片示範的這樣）。第一次建立時，會是綠色的提示字。

建立完成後，我們取得倉庫連結，接著就可以準備將本地的Hexo網站部署上去了。

第二步：手動部署－使用 Hexo 指令發布網站

Hexo內建了非常方便的一鍵部署指令。在執行之前，我們只需要修改一個設定檔，告訴Hexo要將網站部署到哪裡。
首先，要先打開Hexo專案根目錄下的_config.yml檔案，到最下方找到 Deployment 的區塊，並修改成以下內容：

這邊要把<你的帳號>替換成你自己的GitHub帳號。
repo欄位就是你剛剛建立的Repository的HTTPS網址。

設定儲存後，在terminal輸入以下三個指令：

  1. hexo clean (可簡寫為 hexo cl)

    這個指令會清除快取檔案（db.json）和之前已產生的靜態檔案資料夾（public），確保每次部署都是從最乾淨的狀態開始。

  2. hexo generate (可簡寫為 hexo g)

    Hexo 會根據你的文章和主題設定，產生整個網站的靜態HTML、CSS和JavaScript檔案，並放在public的資料夾中。

  3. hexo deploy (可簡寫為 hexo d)

    這個指令會將public資料夾內的所有檔案，push到你在_config.yml中設定好的GitHub Repository與指定的分支。

執行完hexo d後，等待幾分鐘後打開瀏覽器，輸入網址https://<你的帳號>.github.io，就可以看到第一個blog誕生啦~

本篇總結與下一步預告

到目前為止，本篇介紹了如何建立Hexo網站，並透過手動指令將它成功部署到網路上。這個流程雖然簡單，卻是理解網站建置與發布的基本概念。
不過，你可能會發現，每次寫完或修改一篇文章，都得重複輸入hexo clean、generate、deploy這三個指令，久了難免覺得有點繁瑣。
有沒有更聰明、更省事的方法呢？當然有！

下一篇文章，我們將會介紹如何利用GitHub Actions來打造一個自動化部署的流程。
只要你將寫好的文章push到GitHub，後續的建置與部署工作將會全自動完成，不用再癡癡等待hexo慢慢部署啦~ Ow< b
