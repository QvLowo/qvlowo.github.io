Hexo Blog教學(四)：GitHub Actions自動部署教學從手動到全自動的完整流程

發表於2025-09-04|更新於2025-09-17|learning-guides
|總字數:400|閱讀時間:1分鐘|瀏覽量:

前言

上一篇：將你的網站手動部署到 GitHub Pages

在上一篇文章中，我們學會了如何透過hexo的指令hexo clean、hexo generate、hexo deploy來手動部署Hexo Blog。
雖然流程不複雜，但每次更新文章都要重複輸入這些指令，對於長期經營blog來說，很麻煩而且隨著文章內容變多，產生靜態檔的時間也會變得很久。

這篇文章會介紹如何利用GitHub Actions來自動部署Hexo Blog。
只要把文章push到GitHub，系統就會自動完成建置與部署，不需要再手動輸入指令，大幅節省時間，這也是現在目前最常見也很穩定的做法。

什麼是 GitHub Actions？

GitHub Actions 是由Github提供CI/CD的一個服務，只要你在指定的資料夾內，建立workflow文件，它就能在偵測某些事件發生(例如：push)時，自動執行你寫的工作流程腳本。
同時，他也有一個很強大的功能是，你可以設定工作排程，在指定的時間自動執行腳本，我覺得這一點非常的棒。

拿這個系列教學文來說，在部署hexo blog時，我們可以設定一個workflow是：

  1. 當你寫好的文章push到github的遠端倉庫時，workflow會自動觸發。
  2. github actions根據你寫的腳本，自動執行你要他做的工作，像是npm install、hexo g產生靜態檔案等指令。
  3. 最後自動將生成的靜態檔案部署到GitHub Pages。
