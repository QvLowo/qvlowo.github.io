Golang入門教學(一)：為什麼要學Go？從Hello World到你的第一個可執行檔

發表於2025-09-03|更新於2025-09-10|Golang
|總字數:1.7k|閱讀時間:5分鐘|瀏覽量:

一、前言：為什麼學Go?

Go（又稱Golang)是Google於2007年開發的程式語言，相對於Java、Python這些大前輩，是相對新的程式語言小朋友(?)
不過可不能小看它！在短短幾年內，Go已經躍升為後端工程師、雲端開發與區塊鏈開發領域的熱門選擇。

要了解Go的厲害之處，得先知道程式語言的兩大特性：分別是動態直譯語言和強型別編譯語言。

  • 動態直譯語言：程式寫起來簡單也比較靈活，所以開發速度快，所以生產力極佳，缺點是執行速度較慢，比較適合用來寫腳本，不適合長期運行大型系統，例如：Python。
  • 強型別編譯語言：程式碼嚴謹、安全，執行效能高，適合正式環境的大型系統，缺點是開發過程相對複雜、上手門檻高，例如：金融業愛用的Java。

Google工程師不是小朋友要做選擇，他們全都要。為了能更快速地建置更安全、可靠與可擴展的生產系統，go語言就此誕生。

Go 語言的現代優勢

現在打開求職網站，你會發現各家公司都在找具備這些技能的人才：

  • 微服務架構 (Microservices)
  • 雲端原生與部署 (Cloud-Native)
  • 高併發處理 (High Concurrency)
  • 易於擴展的系統 (Scalability)

這些剛好都是 Go 的語言優勢，Go被譽為「為雲端而生」的語言，而不是後來才慢慢支援雲端的語言。這讓它在Kubernetes、Docker等雲端生態系中大放異彩。

除此之外，Go 還是一個跨平台語言。無論你用Windows、Linux還是macOS，Go編譯出來的程式都能直接執行。
甚至更厲害的是：Go可以編譯成獨立的二進位檔，不用在目標系統額外安裝Go環境程式就能直接跑起來！
這大幅簡化了部署流程，這也讓Go成為後端開發者與DevOps工程師的心頭好。

另外，Go也是區塊鏈開發的寵兒，像是Ethereum、Hyperledger等知名專案，都有使用Go來建構其核心元件，完美兼顧了效能、穩定性與開發效率。

二、你的第一個 Go 程式：Hello, World!

每個程式語言的傳統起手式都是Hello, World!，我們就從這裡開始吧！

讓我們來拆解這段程式碼：

  • package main
    宣告這個檔案屬於main套件。
    main在go語言裡是有特別意義的，只有需要被編譯成可執行檔的專案，其入口package才一定是main。

  • import "fmt"
    引入內建函式庫的fmt套件(這是format的縮寫)，它提供了格式化輸出等實用功能。

  • func main()
    宣告一個main函式。
    這代表整個程式的進入點(Entry Point)，也就是Go會從main套件中的main函式開始執行程式。

  • fmt.Println("Hello, World!")
    呼叫fmt套件的Println函式，這個函式是印出文字並自動換行。
    如果也是寫過Java的朋朋們這邊應該不陌生，它就和System.out.Println()的方法非常類似。

你可以在官方提供的 Go Tour上親手試試看。
這個網站不用安裝任何東西，非常適合快速體驗 Go 的語法。
你可以把網站範例的”Hello, 世界”，改成這篇文章的提到的”Hello, World!”，然後按下run就可以看到console的輸出結果。

另外要記得，Go的每個檔案第一行都必須宣告package。
像我們用的fmt套件，原始碼的第一行就是package fmt，只不過它不是用來執行，而是提供給其他程式引入使用的。

三、如何執行你的程式？ go run vs. go build

這裡先寫一個hello.go來示範，寫好hello.go之後，要怎麼讓它跑起來呢？
在Go中，最常用的兩個指令是go run和go build。

  • go run:用於開發階段，快速編譯並執行程式。
  • go build:用於生產部署，會把程式編譯成獨立的可執行檔。

簡單來說，兩者的差異是：

  • go run就好比是你按下IDE(像是IntelliJ IDEA)的播放鍵，程式會直接跑起來，適合開發與測試。
  • go build則是打包程式，像是Java打包會產生.jar檔一樣，go會生成一個二進位檔(例如在macOS/Linux上是沒有副檔名的執行檔，在Windows上則會是.exe)，和.jar檔不同的是，這個檔案不需要額外安裝Go就能獨立執行。

我們用VSCode來示範一下：

  • 使用go run
    直接在terminal輸入指令go run hello.go

    可以看到圖片中console跑出了Hello, World!，代表程式成功被編譯並執行。

  • 使用go build
    直接在terminal輸入指令go build hello.go

    可以看到圖片中左側多了一個叫hello的二進位檔。

  • 執行二進位檔
    我們來執行看看這個檔案會發生什麼事，輸入指令./hello：

    可以發現一樣跑出了Hello, World!，代表我們成功地執行了這個獨立的程式。

四、本篇總結與下一步

到這邊，我們也完成了第一個小小的實作啦~
從為什麼要學 Go開始，寫下每個程式語言必學的第一個Hello, World!，再到用go run、go build執行程式，整個流程應該有點感覺了吧！

快速幫大家複習一下本篇重點：

  • Go的特色：就是語法簡單、效能又快，特別適合雲端、後端、高併發系統。
  • Go 程式結構：package main、import、func main()，是程式啟動的必備三要素。
  • 執行方式：go run適合開發測試；go build則是打包成獨立檔案，丟去哪都能跑， 超方便~

是不是覺得Go的入門比想像中還簡單？不用一堆複雜設定，隨便一個.go檔加幾行程式碼就能跑出結果，這就是Go迷人的地方:開箱即用！
