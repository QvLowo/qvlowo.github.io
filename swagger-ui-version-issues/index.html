【Swagger-ui】Spring Boot與Swagger版本不相容的踩坑紀錄

發表於2025-03-03|更新於2025-11-04|development-practice
|總字數:275|閱讀時間:1分鐘|瀏覽量:

JAVA版本: 17

起因：

以前用 Spring Boot3 + Java17 開發Side Project時，想導入Swagger卻一直失敗。
爬了很多文章後才發現原來要改用 *SpringDoc**才能成功。

最近開發新專案，使用Spring Boot 3.4.2搭配SpringDoc（OpenAPI）2.6.0時，卻又遇到了錯誤訊息：

一開始以為是設定問題，結果實測後發現，果然又是版本不相容的老問題。
簡單來說，問題點是：
SpringBoot版本太新
Swagger版本太舊

解法：提升版本

在 pom.xml 中設定自訂屬性與依賴版本即可。

  1. 自定義properties
  1. 引入Swagger（SpringDoc）依賴

Spring Boot搭配Swagger-ui(SpringDoc)的版本兼容紀錄

Spring Boot 版本 對應SpringDoc版本
3.3.9 2.6.0
3.4.2 2.8.5

升級Spring Boot時，記得同步檢查SpringDoc的版本，不然Swagger UI很容易報錯。
